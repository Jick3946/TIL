### Context
n8n을 통해 TIL 자동화를 시도하면서, 기존의 GPT API 사용으로 인해 비용이 발생하는 문제가 있었습니다. 이를 해결하기 위해 제미나이 API로 변경을 시도했으나, 호출 제한 문제로 대안이 필요했습니다. 결국 Groq이라는 AI API를 사용하는 사이트의 API를 활용하여 문제를 해결했습니다.

### Core
n8n에서 Groq AI API를 통합하기 위해 다음과 같은 설정이 필요합니다:

* Groq API의 API Key를 발급받아 n8n의 인증 정보에 입력합니다.
* HTTP Request 노드를 사용하여 Groq API에 요청을 보냅니다. 
* 결과를 처리하기 위해 필요한 변환 노드를 추가 및 수정합니다.

코드 스니펫 예시:
```javascript
{
  "name": "Groq API Request",
  "type": "n8n-nodes-base.httpRequest",
  "position": [500, 300],
  "parameters": {
    "url": "https://api.groq.ai/perform",
    "method": "POST",
    "body": {
      "key": "사용자 API 키",
      "input": "사용자 요청 데이터"
    }
  }
}
```

### Insight
제미나이 API는 비용 문제를 해결하는 데 한계가 있었습니다. Groq API는 보다 저렴하면서도 유연한 대안을 제공했으며, API Key를 통한 인증으로 사용이 용이했습니다. n8n과의 통합에서는 HTTP Request 노드와 적절한 인증 및 요청 데이터를 사용하는 것이 핵심입니다.
